# EchoFrame Benchmark Evaluation

## Purpose
This benchmark suite is designed to establish a strict, reproducible baseline of the current MNEMOS read path and compare it against the experimental EchoFrame candidate adapter. 

## Scope and Non-Goals
EchoFrame is being evaluated **only** as a candidate read-path adapter (evidence packetization and context compression). 

EchoFrame **must not** alter or assume ownership of:
* MNEMOS ingestion
* Authoritative storage
* Vector or lexical indexing
* Memory lifecycle policy
* Contradiction policy
* Governance doctrine
* Source-of-truth engram records

## Promotion Gates
The EchoFrame prototype will only be considered for deeper integration if it passes the following strict evaluation gates:

* **Gate A: Fidelity** - No material loss in answer correctness.
* **Gate B: Provenance** - Every packeted claim maps back to MNEMOS source evidence.
* **Gate C: Governance Preservation** - Evidence gaps, contradictions, stale flags, and approval-required states survive packetization.
* **Gate D: Token Efficiency** - Meaningful input-token reduction versus the baseline.
* **Gate E: Context Stability** - Repeated/similar queries show lower or equal context thrash.
* **Gate F: Explainability** - Decode decisions and context dropping must be inspectable.

## Qdrant Dimension Mismatch (Phase 1C)
When not loading the `.env.mnemos` configuration, `MnemosConfig` defaults to `BAAI/bge-base-en-v1.5` which produces a 768 dimension vector.
However, the Qdrant collection (`mnemos_engrams`) was initialized using the `all-MiniLM-L6-v2` model which has a dimension of 384.
This mismatch caused Qdrant to reject search queries with `expected dim: 384, got 768`.

To fix this **without destructive reindexing**, the `run_baseline.py` script has been updated to explicitly load `.env.mnemos` to use the correct model dimension (384).

**Important Note:** The current clean baseline (`baseline_results.real.clean.json`) is valid *only* for the `all-MiniLM-L6-v2` / 384-dimension Qdrant profile. If MNEMOS is migrated to `BAAI/bge-base-en-v1.5` or another model, the baseline and candidate runs must be regenerated.

## Running the Benchmark

### Baseline Usage
EchoFrame comparisons must use either:
- **A clean baseline**: `baseline_results.real.clean.json` (Generated by a fully healthy MNEMOS read path)
- **An explicitly accepted degraded baseline**: `baseline_results.real.degraded.json` (with limitations documented)

### 1. Run Real MNEMOS Baseline
To capture the current MNEMOS baseline metrics against the actual read path:
```bash
python benchmarks/echoframe_eval/run_baseline.py
```
This will output `baseline_results.real.clean.json` (if healthy) into the `results/` directory using the `MnemosBaselineAdapter` and the real MNEMOS core runtime.

### 2. Run EchoFrame Candidate
After capturing the baseline, run the prototype packet renderer. You can specify the rendering mode:
```bash
python benchmarks/echoframe_eval/run_candidate.py --mode json_v0
python benchmarks/echoframe_eval/run_candidate.py --mode compact_v0
python benchmarks/echoframe_eval/run_candidate.py --mode compact_safe_v0
```

**Renderer Modes:**
- `json_v0` = verbose structured packet
- `compact_v0` = lean packet, known truncation risk (not promotion-eligible because hardened validation found numeric threshold loss)
- `compact_safe_v0` = fact-preserving compact packet
- `compact_selective_v0` = deterministic query-aware packet
- `compact_semantic_v0` = semantic fact-scored packet, shadow-only candidate
- `compact_semantic_minEvidence_v0` = minimized evidence excerpts with fact-local windows

### 3. Run Comparison
Compare the baseline against the candidate to generate metrics and validation against promotion gates:
```bash
python benchmarks/echoframe_eval/compare_baseline_candidate.py
```
This generates `echoframe_comparison_report.v0.md` and `echoframe_comparison_metrics.v0.json` in the `reports/` and `results/` folders.

### 4. Run Hardened Comparison (Phase 2C)
Stress-test the compact mode against an adversarial set of 40 edge cases:
```bash
python benchmarks/echoframe_eval/compare_hardened.py
```
This produces `echoframe_comparison_report.compact_v0.hardened.md` separating the original and expanded edge cases, alongside strict safety checks for numbers, negations, and exceptions.

### 4. Run Stub Mode (Instrumentation Smoke Test)
If you just want to test the harness without initializing MNEMOS:
```bash
python benchmarks/echoframe_eval/run_baseline.py --use-stub
```
This will output `baseline_results.stub.json`. This is only for debugging the harness and is **not** the baseline.
