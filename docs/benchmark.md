## Track 3: Installer Overhead

**Product question:** *Does the guided installer add value over manual deployment, or just ceremony?*

### What We Tested

The MNEMOS installer (`python -m installer --profile <name>`) automates deployment configuration by:

1. Running host capability probes (GPU, RAM, Docker, NVIDIA runtime)
2. Generating a validated `docker-compose.generated.yml`
3. Generating a `.env.mnemos` with profile-specific variables
4. Generating a `mnemos_profile.yaml` deployment manifest (the deployment contract artifact)

We compared this against a **manual baseline** that copies the compose template directly and writes a minimal `.env` file — the fastest possible manual path.

### Methodology

- **Corpus:** 10,000 synthetic engrams (4 domains: finance, legal, medical, technical)
- **Profiles tested:** `core_memory_appliance`, `governance_native`
- **Runs per profile:** 3 (median reported)
- **Installer mode:** Non-interactive (`--profile` flag), file generation only (no Docker)
- **Validation:** Each run checked for compose, env, and manifest file existence + content correctness

### Results

| Metric | Core Memory Appliance | Governance Native |
|---|---|---|
| Installer time (median) | 0.537s | 0.554s |
| Manual time (median) | 0.001s | 0.001s |
| Installer errors | 0 / 3 | 0 / 3 |
| Manual errors | 0 / 3 | 0 / 3 |
| Validation pass rate | **100%** | **100%** |
| Generates manifest | **Yes** | **Yes** |

### Interpretation

The installer adds ~0.5s of overhead compared to raw file copy. In exchange, it provides:

| Capability | Manual | Installer |
|---|---|---|
| Compose file | Copy from template | Generated from profile spec |
| Environment file | Hand-written | Generated with profile defaults |
| Deployment manifest | **Not generated** | **Auto-generated** (`mnemos_profile.yaml`) |
| Host probe validation | **None** | GPU, RAM, Docker, NVIDIA runtime |
| Reproducibility | Operator-dependent | Deterministic (same inputs = same output) |
| Auditability | None | Manifest captures probe results + profile rationale |

**Conclusion:** The installer's 0.5s overhead is justified. The deployment manifest — which manual deployment cannot produce — is the contract artifact that enables profile migration, supportability, and CI validation. For teams that value deployment discipline, this is non-negotiable infrastructure, not ceremony.
